Pipeline Run Guide
Multi-agent implementation guide — interactive mode

Overview

This guide walks you through running the full Project Caravela data pipeline using 8 sequential/parallel agent invocations. You act as the human orchestrator — reviewing each agent's output before committing and proceeding.

Pipeline Architecture
Phase 1 (sequential):  1a Meltano → 1b Staging → 1c Marts → 1d Tests
                                                                    ↓
Phase 2 (parallel):                              2 Dagster ‖ 3 Notebooks
                                                                    ↓ (Agent 3 done)
Phase 3:                                                    4 Dashboard
                                                                    ↓
Phase 4:                                                    5 Exec Brief
                                                                    ↓
                                                          Final Cleanup

Before You Start

1. GCP Service Account

Create a service account with these roles:

  • BigQuery Data Editor
  • BigQuery Job User

Download the JSON key and store it somewhere safe (e.g. ~/.gcp/caravela-sa.json).

2. Set Environment Variables

Add these to your shell profile (~/.zshrc) or export before each session:

export GOOGLE_APPLICATION_CREDENTIALS="$HOME/.gcp/caravela-sa.json"
export GCP_PROJECT_ID="your-gcp-project-id"

3. Activate conda environment

conda activate assignment2

4. Create BigQuery datasets (one-time)

bq mk --dataset $GCP_PROJECT_ID:olist_raw
bq mk --dataset $GCP_PROJECT_ID:olist_analytics

Or use the --create-datasets flag in the pre-check script.

⚠ Quota Warning The pipeline runs 8 agent sessions. Each session consumes API tokens. If your weekly quota runs out mid-agent, the session will end. The git history and progress.md are your recovery points — they survive session loss. When quota refreshes, re-run the agent that was interrupted (all dbt models are idempotent).

Troubleshooting

meltano install fails

Symptom: pkg_resources ImportError during meltano install

Cause: setuptools v81 removed pkg_resources.

Fix: Ensure both pip_url entries in meltano.yml include setuptools<70. If meltano install was already run without the fix, delete .meltano/ and re-run:

rm -rf meltano/.meltano
cd meltano && meltano install

dbt build: table not found in olist_raw

Symptom: dbt models fail with "Table not found" when referencing olist_raw tables

Cause: Meltano has not been run yet — olist_raw tables don't exist.

Fix: Run Meltano first to load the source data, then re-run dbt build:

cd meltano && meltano run tap-spreadsheets-anywhere target-bigquery

Note: the geolocation load (1M rows) takes several minutes. Do not interrupt.

dagster dev fails: manifest.json not found

Symptom: FileNotFoundError: manifest.json on dagster dev

Fix: Run dbt parse first (no BigQuery connection needed):

cd dbt && dbt parse

Choropleth map is blank

Symptom: Geographic choropleth renders with no fill colours

Most likely cause: featureidkey is wrong. Must be properties.sigla.

Check: Confirm the locations column contains 2-letter state codes (e.g. SP, RJ) — not full names.

Agent quota exhaustion mid-session

What happened: The Claude session ended unexpectedly due to API quota limits.

Recovery steps:

  1. Check git log --oneline to see what was committed before the session ended
  2. Check worktrees/agent-X/ for any uncommitted partial work
  3. When quota refreshes, re-run ./scripts/launch_agent.sh <id> for the interrupted agent
  4. All dbt models are idempotent — re-running overwrites cleanly

git merge conflict on wrap-up

Symptom: wrap_up_agent.sh fails with merge conflict

Most likely cause: Two parallel agents (2 and 3) both modified a shared file (unlikely by design, but possible if an agent strayed outside its directory).

Fix: Resolve manually in the worktree:

cd worktrees/agent-2  # or agent-3
git status            # see conflicting files
# resolve conflicts manually
git add -A
git commit -m "Resolve merge conflict"
cd ../..
git merge worktrees/agent-2

notebooks/utils.py import fails

Symptom: Dashboard or generate_parquet.py fails with ImportError on utils

Fix: Run from repo root (not from notebooks/):

cd /path/to/Project_Caravela
python -c "from notebooks.utils import *"

If the import still fails, check that notebooks/__init__.py exists (create an empty one if missing).
